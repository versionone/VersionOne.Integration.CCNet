CruiseControl.NET : VersionOne Publisher
This page last changed on Mar 03, 2008 by dhanson.

The VersionOne Publisher is used to create Build Runs in a VersionOne System. These Build Runs correspond to a single build output for a single Project in CruiseControl.NET.

You may place the <versionone /> in the <publishers /> section in your Project Configuration Block.

Examples

Basic Example:

<versionone url="http://server/VersionOne/" username="admin" password="admin" />

Basic Example with WebRoot:

<versionone webroot="http://server/ccnet/" url="http://server/VersionOne/" username="admin" password="admin" />

NT Auth Example:

<versionone url="http://server/VersionOne/" integratedauth="true" />

NT Auth Example with Username/Password:

<versionone url="http://server/VersionOne/" username="user@domain" password="password" integratedauth="true" />

Basic Example with ChangeSet Updates:

<versionone url="http://server/VersionOne/" username="admin" password="admin" referenceexpression="[A-Z]{1,2}-[0-9]+" referencefield="Number" />

Configuration Elements:

Note: Because the publisher can be configured with or without NT Auth, not all elements are marked as required. See the descriptions below for more information.
Node Description Type Required Default
url The URL of the VersionOne Application. For example: http://server/VersionOne/ string true  
username The username to authenticate with. This field is optional if using NT Auth (integrated auth). Required when using VersionOne authentication. string (varies)  
password The password to authenticate with. This field is optional if using NT Auth (integrated auth). Required when using VersionOne authentication. string (varies)  
integratedauth True to use NT Auth (integrated auth), false to use VersionOne authentication. boolean (varies) false
webroot The root of the URL used for linking to build logs. For example: http://localhost/ccnet/ string false null
server The "BuildServer" portion of the URL used for linking to build logs. For example: http://localhost/ccnet/server/{BuildServer}/project/... string false "local"
referenceexpression The regular expression to use when matching primary workitems (stories and defects) with change comments. Required when using changeset integration. string (varies) null
referencefield The system name of an attribute to search when matching primary workitems (stories and defects) with change comments. Required when using changeset integration. string (varies) null
